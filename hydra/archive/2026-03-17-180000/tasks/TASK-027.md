## Description

Wire the generated mapping methods into `GeneratedMapRegistry` via `[ModuleInitializer]`.

### Generated Code Shape

```csharp
// <auto-generated />
#nullable enable
using System.Runtime.CompilerServices;
using DeltaMapper.Runtime;

namespace UserProject
{
    public partial class UserProfile
    {
        private static void Map_User_To_UserDto(User src, UserDto dst)
        {
            dst.Id = src.Id;
            dst.Name = src.Name;
            dst.Email = src.Email;
        }

        [ModuleInitializer]
        internal static void RegisterGeneratedMaps_UserProfile()
        {
            GeneratedMapRegistry.Register<User, UserDto>(static (src, dst) =>
                Map_User_To_UserDto(src, dst));
        }
    }
}
```

### Key Design Points

1. Method name includes the profile class name to avoid conflicts when multiple profiles exist
2. `static` lambda to avoid closure allocation
3. One `[ModuleInitializer]` method per profile class (registers all mappings from that profile)
4. For multiple `[GenerateMap]` on one class, the initializer registers all of them
5. Uses fully qualified `System.Runtime.CompilerServices.ModuleInitializer` to avoid using conflicts

### EmitHelper Changes

Add methods to emit:
- The `using` statements for `System.Runtime.CompilerServices` and `DeltaMapper.Runtime`
- The `[ModuleInitializer]` method body
- Multiple `Register` calls when a class has multiple `[GenerateMap]` attributes
