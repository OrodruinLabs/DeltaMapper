# EF Core Integration

## Overview

`DeltaMapper.EFCore` provides middleware that detects Castle.Core dynamic proxies emitted by EF Core's lazy loading infrastructure. This enables proxy-aware mapping behavior in the pipeline.

## Install

```bash
dotnet add package DeltaMapper.EFCore
```

## Usage

```csharp
var config = MapperConfiguration.Create(cfg =>
{
    cfg.AddProfile<UserProfile>();
    cfg.AddEFCoreSupport();
});

IMapper mapper = config.CreateMapper();
var dto = mapper.Map<User, UserDto>(trackedEntity);
```

`AddEFCoreSupport()` is an extension on `MapperConfigurationBuilder` and returns the builder for fluent chaining.

## How It Works

The `EFCoreProxyMiddleware` inspects the source object's runtime type to detect Castle.Core dynamic proxy classes generated by EF Core's `UseLazyLoadingProxies()`. This detection is the foundation for proxy-aware mapping — the middleware currently identifies proxies and passes through, providing the hook point for future navigation-skipping logic.

For non-proxy entities, the middleware is a pass-through with negligible overhead.

## Current Limitations

- Navigation property skipping based on load state is not yet implemented — the middleware detects proxies but does not conditionally skip properties
- The InMemory provider does not generate Castle.Core proxies, so proxy detection requires a relational provider (e.g., SQLite) with `UseLazyLoadingProxies()`
