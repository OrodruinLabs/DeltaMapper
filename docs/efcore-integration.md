# EF Core Integration

## Overview

`DeltaMapper.EFCore` provides two capabilities for EF Core applications:

1. **Proxy-aware mapping** — middleware that detects Castle.Core dynamic proxies emitted by EF Core's lazy loading infrastructure and prevents accidental lazy-load triggers during in-process mapping.
2. **`ProjectTo<T>()` projections** — translates profile mapping configurations into `Expression` trees that EF Core converts to SQL, so only the columns needed for the destination type are fetched.

## Install

```bash
dotnet add package DeltaMapper.EFCore
```

---

## Proxy Detection

### Usage

```csharp
var config = MapperConfiguration.Create(cfg =>
{
    cfg.AddProfile<UserProfile>();
    cfg.AddEFCoreSupport();
});

IMapper mapper = config.CreateMapper();
var dto = mapper.Map<User, UserDto>(trackedEntity);
```

`AddEFCoreSupport()` is an extension on `MapperConfigurationBuilder` and returns the builder for fluent chaining.

### How It Works

The `EFCoreProxyMiddleware` inspects the source object's runtime type to detect Castle.Core dynamic proxy classes generated by EF Core's `UseLazyLoadingProxies()`.

When a proxy is detected, the middleware skips all collection-typed properties during mapping. This prevents accidental lazy loading triggers that would otherwise issue extra database queries or throw if the `DbContext` has already been disposed. The proxy flag is scoped per source object, so nested non-proxy objects retain normal collection mapping.

Scalar properties are mapped normally. Non-proxy entities pass through the middleware unchanged with negligible overhead.

> **Note:** The middleware only skips *collection* properties. Reference navigation properties (single-object navigations like `Order.Customer`) are **not** skipped and may still trigger lazy loading on proxy entities if not eagerly loaded. Use `.Include()` for any reference navigations you need mapped, or project to a DTO that excludes them.

### Limitations

- The InMemory provider does not generate Castle.Core proxies, so proxy detection requires a relational provider (e.g., SQLite, SQL Server) with `UseLazyLoadingProxies()`
- Eagerly-loaded single-entity navigations on proxy types are always mapped; only collection navigations are skipped

## ProjectTo — IQueryable Projections

`ProjectTo<TDst>()` translates profile mapping configuration into an `Expression<Func<TSrc, TDst>>` that EF Core's LINQ provider converts to SQL. Only the columns referenced by the destination type are fetched — no full entity hydration.

### Usage

```csharp
var config = MapperConfiguration.Create(cfg => cfg.AddProfile<OrderProfile>());

// Strongly-typed source overload
var dtos = await dbContext.Orders
    .Where(o => o.IsActive)
    .ProjectTo<Order, OrderDto>(config)
    .ToListAsync();

// Non-generic source overload (when source type is only known as IQueryable)
IQueryable query = dbContext.Orders.Where(o => o.IsActive);
var dtos = await query
    .ProjectTo<OrderDto>(config)
    .ToListAsync();
```

### API

```csharp
namespace DeltaMapper.EFCore;

public static class QueryableExtensions
{
    // Strongly-typed source — preferred when TSrc is known at compile time
    public static IQueryable<TDst> ProjectTo<TSrc, TDst>(
        this IQueryable<TSrc> source, MapperConfiguration config);

    // Non-generic source — use when source is IQueryable without a type parameter
    public static IQueryable<TDst> ProjectTo<TDst>(
        this IQueryable source, MapperConfiguration config);
}
```

### Supported Features

| Feature | Supported |
|---|---|
| Convention name matching | Yes |
| `ForMember` / `MapFrom` | Yes |
| `Ignore` | Yes |
| `NullSubstitute` (null-coalesce) | Yes |
| Flattening (`Order.Customer.Name` → `CustomerName`) | Yes |
| Nested object navigation | Yes |
| Collection navigation (`.Select().ToList()`) | Yes |
| `BeforeMap` / `AfterMap` | No — throws `DeltaMapperException` |
| `ConstructUsing` | No — throws `DeltaMapperException` |
| `Condition` | No — throws `DeltaMapperException` |

### Limitations

- `BeforeMap`, `AfterMap`, `ConstructUsing`, and `Condition` cannot be expressed as pure expression trees. Calling `ProjectTo` on a map that uses any of these throws `DeltaMapperException` at the point the expression is built (before query execution).
- `ProjectTo` does not go through the middleware pipeline — it generates a SQL projection, not an in-process mapping call. `AddEFCoreSupport()` is not required when using `ProjectTo` exclusively.
